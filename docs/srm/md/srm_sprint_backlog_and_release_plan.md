# SRM Sprint Backlog and Release Plan

## Release Plan

### R1 Foundation
Epics:
- supplier master
- directory
- contacts
- categories
- permissions

Outcome:
Usable supplier directory and 360 with correct visibility controls.

### R2 Onboarding
Epics:
- application intake
- onboarding checklist
- document requests
- approval workflow

Outcome:
New suppliers can be created and approved in a governed way.

### R3 Compliance
Epics:
- document repository
- expiry logic
- screening/check records
- suspension rules

Outcome:
Compliance health visible and actionable.

### R4 Performance / Risk
Epics:
- KPIs
- scorecards
- incidents
- risk assessments
- review workspace

Outcome:
Structured supplier evaluation live.

### R5 Governance
Epics:
- QBRs
- action plans
- audit depth
- change approvals
- exports

Outcome:
Governed supplier management with improvement loops.

### R6 Integrations
Epics:
- control tower
- finance
- spend feeds
- future tender/tariff readiness

Outcome:
SRM connected to surrounding platform.

## Cross-Cutting Acceptance Criteria
- No supplier can be activated without mandatory onboarding steps complete.
- Critical document expiry creates visible alerts and defined workflow.
- Scorecards retain period history rather than overwriting.
- Major master-data changes are auditable and approval-controlled.
- Supplier category/region qualification can restrict where supplier is usable.
