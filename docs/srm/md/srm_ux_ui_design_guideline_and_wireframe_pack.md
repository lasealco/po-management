# SRM UX / UI Design Guideline and Wireframe Pack

## UX Principles
- Supplier directory should be filter-rich and score-aware.
- Supplier 360 should present status, compliance health, scorecards, and next actions immediately.
- Expiring documents and overdue reviews must be visible early.
- Use tabs and summary cards to separate details, performance, risk, and documents.
- Keep approval workflows obvious and auditable.

## Wireframe Coverage

### Supplier Directory
Primary user:
- SRM analyst / manager

Main blocks:
- global filters
- directory grid
- status chips
- risk/performance columns

Key actions:
- open supplier
- bulk request docs
- export view

### Supplier 360
Primary user:
- cross-functional

Main blocks:

- tabs
- alert panel

- suspend

Primary user:

## Textual Wireframes
- Supplier Directory: top filters; chips for Active / Conditional / Suspended / Expiring Docs; main grid with score, risk, service types, owner; bulk actions.
- Supplier 360: header with status, tier, risk, performance, compliance, last review; tabs for profile, contacts, capabilities, qualification, compliance, contracts, performance, risk, relationship, documents, tasks, audit.
- Onboarding Workspace: left checklist, center document and questionnaire panel, right approval history and comments.
