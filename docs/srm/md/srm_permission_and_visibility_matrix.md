# SRM Permission and Visibility Matrix

## Role Rights

### Read-Only Business User
- Directory: Yes
- Create/Edit Master: No
- Approve Onboarding: No
- See Risk: Limited
- See Commercial: No or limited
- Upload Docs: No
- Reviews/Scorecards: View only
- Admin: No

### SRM Analyst
- Directory: Yes
- Create/Edit Master: Yes
- Approve Onboarding: No
- See Risk: Yes
- See Commercial: Limited
- Upload Docs: Yes
- Reviews/Scorecards: Yes
- Admin: No

### Compliance User
- Directory: Yes
- Create/Edit Master: Limited
- Approve Onboarding: Yes for compliance steps
- See Risk: Full
- See Commercial: No or limited
- Upload Docs: Yes
- Reviews/Scorecards: Limited
- Admin: No

### Operations Manager
- Directory: Yes
- Create/Edit Master: Limited
- Approve Onboarding: Yes for operational scope
- See Risk: Yes
- See Commercial: Limited
- Upload Docs: Yes
- Reviews/Scorecards: Yes
- Admin: No

### Commercial / Procurement Manager
- Directory: Yes
- Create/Edit Master: Yes
- Approve Onboarding: Yes
- See Risk: Yes
- See Commercial: Full
- Upload Docs: Yes
- Reviews/Scorecards: Yes
- Admin: No

### Executive / Head
- Directory: Yes
- Create/Edit Master: Limited
- Approve Onboarding: Yes
- See Risk: Full
- See Commercial: Full
- Upload Docs: View
- Reviews/Scorecards: Full
- Admin: No

### System Admin
- Directory: Config
- Create/Edit Master: Config
- Approve Onboarding: Config
- See Risk: Config
- See Commercial: Config
- Upload Docs: Config
- Reviews/Scorecards: Config
- Admin: Yes

## Sensitive Fields
- Bank details: approval and limited visibility
- Insurance and compliance failure notes: restricted by role
- Commercial terms summary: restricted to commercial/procurement/management
- Risk details and mitigations: restricted by role
- Internal review commentary: restricted by role
- Suspension rationale: restricted by role
